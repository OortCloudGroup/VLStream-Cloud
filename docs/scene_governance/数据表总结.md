# 场景治理数据表总结

## 🎯 创建目的

根据场景治理前端页面的**列表页字段**和**编辑页面字段**，设计并创建了 `scene_governance` 数据表，用于存储和管理AI决策场景的配置信息。

## 📋 字段来源分析

### 列表页字段来源
- **序号** → `id` (主键)
- **场景名称** → `name` 
- **场景描述** → `description`
- **关联设备** → `devices`
- **治理规则** → `rules`
- **状态** → `status`
- **创建时间** → `created_at`

### 编辑页字段来源
- **场景名称** → `name`
- **执行时间** → `execute_type`
- **选择天数** → `selected_days` (JSON)
- **间隔** → `interval_num`
- **AI算法** → `algorithm`
- **区划地点** → `location`
- **摄像头** → `cameras`
- **开始时间** → `start_time`
- **结束时间** → `end_time`

### 新增弹窗字段
- **名称** → `name`
- **备注** → `description`

## 📊 数据表特点

### 1. 核心字段设计
- **主键**: `id` - 自增长整型
- **名称**: `name` - 支持中文，最大100字符
- **描述**: `description` - 使用TEXT类型支持长文本
- **状态**: `status` - 枚举类型，支持启用/禁用
- **执行类型**: `execute_type` - 支持每天/隔天/每周/每月

### 2. 特殊字段设计
- **选择天数**: `selected_days` - 使用JSON数组存储
- **时间字段**: 支持开始时间、结束时间、创建时间、更新时间
- **软删除**: `deleted_at` - 支持软删除功能
- **审计字段**: `created_by`, `updated_by` - 记录操作人

### 3. 索引和约束
- **主键索引**: `id`
- **查询索引**: `name`, `status`, `execute_type`, `created_at`
- **软删除索引**: `deleted_at`
- **检查约束**: 状态值、执行类型、间隔数量的有效性

## 🛠️ 技术实现

### 数据库引擎
- **存储引擎**: InnoDB
- **字符集**: utf8mb4 (支持中文和特殊字符)
- **版本要求**: MySQL 5.7+ (支持JSON数据类型)

### JSON字段设计
```json
{
  "selected_days": ["monday", "wednesday", "friday"]
}
```

### 软删除机制
- `deleted_at` 为 NULL: 记录有效
- `deleted_at` 不为 NULL: 记录已删除

## 📈 使用场景

### 1. 前端列表页查询
```sql
SELECT id, name, description, devices, rules, status, created_at
FROM scene_governance 
WHERE deleted_at IS NULL
ORDER BY created_at DESC;
```

### 2. 编辑页面数据加载
```sql
SELECT * FROM scene_governance 
WHERE id = ? AND deleted_at IS NULL;
```

### 3. 状态筛选
```sql
SELECT * FROM scene_governance 
WHERE status = 'enabled' AND deleted_at IS NULL;
```

### 4. 按执行类型筛选
```sql
SELECT * FROM scene_governance 
WHERE execute_type = 'weekly' AND deleted_at IS NULL;
```

## 📝 注意事项

1. **JSON字段查询**: 使用 `JSON_CONTAINS` 函数查询JSON数组
2. **软删除**: 所有查询都需要加上 `deleted_at IS NULL` 条件
3. **时间字段**: 使用 `DATETIME` 类型，注意时区问题
4. **字符集**: 确保数据库连接使用 `utf8mb4` 字符集
5. **事务处理**: 重要操作使用事务保证数据一致性
